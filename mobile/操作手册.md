# 🤖 WALL-E 手机版 · 操作手册

> **开发状态：** 手机移动端 **尚未开发完成**，界面与功能仍在迭代，与 Windows / macOS 桌面版存在差距，**暂不作为正式对外发布版本**。日常请使用桌面版；本文档供内部测试与打包参考。

> 以电影《机器人总动员》主角 **WALL-E（瓦力）** 为原型的 **Android 伴侣应用**。
> 采用与桌面版相同的像素风动画资源，在手机上陪你专注、休息，并管理待办。
>
> 本应用为独立 **Kivy** 项目，与桌面版（PySide6）**不是同一个安装包**；核心待办/记事/提醒/番茄钟与 CloudBase 同步已与桌面对齐，但无桌面悬浮宠物与键鼠联动。

## 一、功能总览

| 功能 | 说明 |
| --- | --- |
| 📋 待办清单 | 个人待办、派发四子页、三级优先级、时间显示、归档 |
| 📝 记事本 | 多条备忘，编辑后自动保存 |
| ⏰ 定时提醒 | 单次/每天/工作日等；到点系统通知 |
| ⏱️ 番茄钟 | 可配置工作/休息/轮数；后台继续计时 |
| ☁️ 账号同步 | 授权码 + 手机号登录；与 Windows / macOS 同步 |
| 📤 任务派发 | **派给我的 / 我派出的**；分区展示；通知可跳转子页 |
| 📱 竖屏应用 | 底部导航：**待办 · 记事 · 提醒 · 番茄钟 · 账号** |

**不包含（桌面独有）：** 桌面悬浮瓦力、键鼠联动动画、休息全屏遮罩、系统托盘、中英文界面切换。

## 二、与桌面版的区别

| 项目 | 手机版（Android） | 桌面版（Windows / macOS） |
| --- | --- | --- |
| 运行环境 | APK，Android 7.0+（API 24+） | exe / msi（Win）；dmg / app（Mac） |
| 界面语言 | 中文为主 | 简体中文 / English 可切换 |
| 瓦力展示 | ❌ 无桌面悬浮宠物 | 桌面悬浮窗 + 信封/旗子徽章 |
| 待办 | 四子页 + 派发分区，与桌面一致 | 瓦力桌面助手 + 电灯泡 + 信封/旗子 |
| 记事 / 提醒 | ✅ | ✅ |
| 派发通知 | 系统通知，可跳转待办子页 | 瓦力气泡 |
| 账号 / 同步 | ✅ 授权码 + 手机号；可暂停自动同步 | ✅ 同左 |

启用同步后，待办/记事/提醒/番茄钟设置可在 **Windows ↔ macOS ↔ Android** 间自动合并。详见 [GUIDE/sync/CLOUDBASE_SETUP.md](../GUIDE/sync/CLOUDBASE_SETUP.md) 与桌面 [操作手册.md](../操作手册.md) 第十六节。

## 三、安装到手机

### 方式 A：安装调试 APK（自行打包或他人分发）

1. 取得 APK 文件（常见文件名示例：`walle-1.0.0-arm64-v8a-debug.apk`）。
2. 将 APK 复制到手机（数据线、网盘、微信文件助手等均可）。
3. 在手机**文件管理器**中点击该 APK。
4. 若系统提示「禁止安装未知应用」，到 **设置 → 安全 / 应用安装** 中，为当前文件管理器或浏览器**允许安装未知来源**。
5. 按提示完成安装，桌面会出现 **WALL-E** 图标。

> **架构说明：** 打包脚本可能生成 `arm64-v8a`（多数近年手机）或 `armeabi-v7a`（较老 32 位机）版本。装错架构会提示无法安装，请换对应 APK 重试。

### 方式 B：通过电脑 adb 安装

手机开启**开发者选项 → USB 调试**，连接电脑后执行：

```bash
adb install path/to/walle-1.0.0-arm64-v8a-debug.apk
```

### 卸载

与卸载普通应用相同：**设置 → 应用 → WALL-E → 卸载**。  
卸载后应用私有目录内的待办数据会一并清除。

## 四、界面与导航

打开应用后，主界面分为上下两部分：

- **上方**：当前页面（**待办** / **记事** / **提醒** / **番茄钟** / **账号**）。
- **下方**：底部导航 — **待办** · **记事** · **提醒** · **番茄钟** · **账号**。

默认进入 **待办** 页；派发相关分段在待办页顶部切换。

## 五、待办清单

### 子页与同步

- 顶部分段：**个人待办** · **派给我的** · **我派出** · **归档**（与桌面命名一致；归档页为已完成任务）。
- 登录后，分段上方显示 **同步状态** 与 **重试同步**。

### 个人待办

1. 在 **个人待办** 子页顶部输入任务，选择优先级，点 **添加** 或回车。
2. 卡片显示 **添加时间**；未完成任务可改 **优先级**下拉。
3. **完成/还原**、**删除** 管理任务；已完成内容在 **归档** 子页查看。

### 派发（须登录）

- **派给我的** / **我派出**：按状态分区（尚未接受、已接受、已回退/退回、已撤回）。
- **我派出** 可填写手机号/昵称、标题、可选 **任务说明** 后派发；可 **撤回**（须理由）。
- **派给我的** 可 **接受/拒绝/完成**；已回退或已撤回可 **清除** 单条。
- 接受后 **不会** 写入个人待办。

### 归档

- 个人与派发已完成任务按 **完成日期** 分组。
- 点 **清除已归档** 清空全部归档；个人任务可 **还原**。

### 首次使用示例

第一次安装会自动创建三条示例待办（开会 / 交周报 / 取快递），可直接完成或删除。

## 八、番茄钟

切到 **番茄钟** 页，中央为大号倒计时 **分:秒**。

| 按钮 | 作用 |
| --- | --- |
| **开始 50 分钟** | 进入专注倒计时（状态显示「专注中…」） |
| **休息 10 分钟** | 进入休息倒计时（状态显示「休息中 ☕」） |
| **停止** | 结束当前计时，状态变为「已停止」 |

倒计时归零后显示 **「时间到！」**，计时自动停止。

**与桌面版的差异：**

- 番茄钟参数可在 **番茄钟** 页配置（工作/休息/轮数），与桌面同步。
- 无「每轮结束前 2 分钟预警」、无全屏休息遮罩、无桌面瓦力气泡。
- 后台/锁屏时通过前台 Service 尽量保持计时；若系统杀死进程，进度可能丢失。

## 九、账号与 CloudBase 同步

切到 **账号** 页：

1. 填写 **授权码**（与 Windows / macOS 相同）→ **保存授权码**。
2. 使用 **密码登录**、**验证码登录** 或 **新用户注册**（手机号 + 密码 + 短信）。
3. 登录成功后，待办/记事/提醒/番茄钟设置与桌面自动合并。
4. **立即同步**、**暂停自动同步**、**退出登录**；可管理联系人昵称（派发时可用昵称）。

派发与待办子页说明见 [第五节](#五待办清单)。部署与故障排查见 [GUIDE/sync/CLOUDBASE_SETUP.md](../GUIDE/sync/CLOUDBASE_SETUP.md)。

## 十、数据保存与备份

### 保存内容

**待办、记事、提醒、番茄钟设置** 在登录后会与 CloudBase 同步；未登录时仅保存在本机。

### 保存位置

| 环境 | 路径 |
| --- | --- |
| Android 真机 | 应用私有目录（含 `todos.json`、`auth.json`、`sync_meta.json` 等） |
| 电脑调试 | `~/.walle-mobile/` 或环境变量 `WALLE_MOBILE_DATA` |

与 Windows `%APPDATA%\WALL-E` **不会自动复制文件**；请用 **相同授权码 + 同一手机号登录** 实现同步。

### 备份建议

- 重要待办请定期在应用内自行核对，或请熟悉 adb 的用户从私有目录导出 `todos.json`。
- 换机前若无导出，卸载旧机应用后数据无法恢复。

## 十一、常见问题 FAQ

**Q：和电脑上的 WALL-E 是同一个软件吗？**  
A：同源项目，Android 为独立 APK。填写相同 **授权码** 并用 **同一手机号** 登录后，可与 **Windows / macOS** 桌面版同步待办等数据。

**Q：待办会同步到电脑吗？**  
A：会。各端填写 **相同授权码**，用 **同一手机号** 登录即可。派发任务需双方各登录至少一次。

**Q：安装时提示「应用未安装」？**  
A：检查 APK 是否损坏、架构是否匹配（arm64 / armeabi-v7a）；若曾安装过同包名不同签名版本，需先卸载旧版再装。

**Q：允许「未知来源」安全吗？**  
A：调试 APK 仅供测试。只安装你信任的来源；上架应用商店需使用正式签名的 release 包。

**Q：番茄钟后台还在走吗？**  
A：有提醒或番茄钟运行时会尽量保持前台 Service；若系统杀死进程，计时可能中断。

**Q：有桌面瓦力宠物吗？**  
A：手机版无桌面悬浮宠物；派发状态通过 **系统通知** 提示，并可跳转待办子页。

**Q：通知权限有什么用？**  
A：用于 **定时提醒到点** 与 **派发任务状态变更** 通知；首次启动请允许通知权限。

## 十二、开发者：打包与调试

普通用户只需阅读上文第三节安装 APK 即可。若你需要**自己编译 APK** 或在 Windows 上**预览界面**，请参阅同目录下的 **[BUILD_ANDROID.md](./BUILD_ANDROID.md)**，内容包括：

- Windows 本地 Kivy 预览（`prepare_assets.py` + `main.py`）
- WSL2 + Buildozer 打包流程
- 虚拟环境与常见报错处理
- 调试 APK 输出路径：`mobile/bin/*.apk`
- **`.buildozer` 构建缓存**：可完全删除以释放空间，下次打包会重新编译（详见 [GUIDE/BUILD_ARTIFACTS.md](../GUIDE/BUILD_ARTIFACTS.md)）

项目根目录可用 `clean_build_artifacts.bat` 一键清理缓存（保留 `dist` 安装包）。

简要预览命令（在项目根目录，需已安装 Python 与 Kivy）：

```powershell
cd mobile
..\.venv\Scripts\python.exe prepare_assets.py
..\.venv\Scripts\python.exe main.py
```
